Für die Umsetzung der beiden Funnel-Varianten wurde ein moderner und 
skalierbarer Tech-Stack gewählt, der sowohl eine effiziente Entwicklung als auch 
eine tiefgründige Analyse des Nutzerverhaltens ermöglicht. Die Frontend-Basis der 
Website besteht aus den Standard-Webtechnologien \textit{HTML}, \textit{CSS} und \textit{JavaScript}. Diese 
bilden die Grundlage für die Struktur, das Design und die Interaktivität der 
Benutzeroberfläche und ermöglichen eine plattformunabhängige Nutzung auf 
unterschiedlichen Endgeräten.

Als zentrales Tool für die Entwicklung und das UI-Design wird \textit{Framer} 
eingesetzt. \textit{Framer} kombiniert visuelles Design mit funktionaler Umsetzung 
und erlaubt die Erstellung interaktiver, produktionsreifer Websites. 
Dadurch können Designentscheidungen direkt 
umgesetzt und optimiert werden, insbesondere im Hinblick auf User Experience 
und Conversion-Optimierung.

Das Hosting der Websites erfolgt über AWS (Amazon Web Services), 
das in \textit{Framer} integriert ist und eine unkomplizierte Veröffentlichung 
sowie Verwaltung der Websites ermöglicht. Durch die Nutzung 
von \textit{Amazon Web Services} wird eine hohe Verfügbarkeit
und eine stabile Performance gewährleistet.

Zur Analyse des Nutzerverhaltens und zur Auswertung der Funnel-Performance 
werden zwei Tracking-Tools eingesetzt.
Zum einen wird \textit{Google Analytics} verwendet, um umfassende Einblicke in das 
Nutzerverhalten zu erhalten.
Zum anderen wird \textit{Microsoft Clarity} eingesetzt, um detaillierte Informationen 
und Screencasts von Nutzerinteraktionen zu sammeln.
Die eingesetzten Tools ermöglichen die Erfassung von Kennzahlen wie Seitenaufrufen,
Verweildauer, Absprungraten sowie Conversion-Raten. Auf dieser Grundlage können 
Unterschiede zwischen den beiden
Funnel-Varianten objektiv bewertet werden.

Für die Umsetzung des dialogbasierten Chatbots wird \textit{Voiceflow} verwendet. 
\textit{Voiceflow} ist eine Plattform zur Entwicklung von Chatbots, 
die es ermöglicht, strukturierte Dialogflüsse zu definieren und in 
bestehende Webanwendungen zu integrieren. Der Chatbot übernimmt dabei die Funktion 
der Informationsvermittlung und Nutzerführung, indem er individuell auf Eingaben 
reagiert und den Entscheidungsprozess interaktiv gestaltet.

Durch die Kombination dieser Technologien entsteht ein umfassender
Tech-Stack, der sowohl die Entwicklung zweier unterschiedlicher Funnel-Ansätze 
als auch deren Analyse im Rahmen dieser Arbeit ermöglicht.

\setauthor{\firstauthor}
\section{Framer}
Framer ist ein No-Code-Tool zur Erstellung und zum Hosting interaktiver Websites.
Es ermöglicht die effiziente Entwicklung von interaktiven und responsiven Benutzeroberflächen.

Eine weitere Funktion von Framer ist die Möglichkeit, dass mehrere
Personen gleichzeitig an einer Website arbeiten können, was 
die Zusammenarbeit innerhalb eines Teams erleichtert.

Über die \textit{Custom-Code}-Funktion von Framer ist es möglich,
eigenen Code in die Website zu integrieren, um zusätzliche Funktionen, 
wie etwa die Einbindung eines Chatbots oder die Erfassung von Nutzerdaten, zu 
realisieren.

\setauthor{\firstauthor}
\subsection{Hosting mit Framer}
Framer bietet die Möglichkeit, Websites direkt über die Plattform zu hosten.
Das interne Hosting erfolgt über Server von AWS (Amazon Web Services), 
wobei die Website unter einer Subdomain der Form \textit{framer.app} bereitgestellt wird.
Alternativ besteht die Möglichkeit, die Website auf einem eigenen Server zu betreiben.

Durch den integrierten Hosting-Service können Änderungen innerhalb weniger Sekunden
veröffentlicht werden, was den Entwicklungsprozess deutlich beschleunigt.
Zudem wird jede veröffentlichte Änderung automatisch versioniert,
wodurch frühere Zustände bei Bedarf einfach wiederhergestellt werden können.

Darüber hinaus ist das Hosting über Framer bis zu einer Bandbreite von
1 Gigabyte pro Monat kostenlos,
während beim Betrieb auf einem eigenen Server zusätzliche Kosten entstehen.

\setauthor{\firstauthor}
\section{Alternativen zu Framer}
Mögliche Alternativen zu Framer sind Dienste wie Webflow oder Wix,
die ebenfalls die Erstellung interaktiver Websites ermöglichen.
Diese Plattformen bieten vergleichbare Funktionen, 
wie beispielsweise die Integration von eigenem Code sowie Hosting-Lösungen.

Dennoch weist Framer in einigen Bereichen Vorteile gegenüber den genannten
Alternativen auf. Dazu zählen unter anderem:
\begin{itemize}
    \item Die Möglichkeit, Änderungen an der Website innerhalb 
    weniger Sekunden zu veröffentlichen, was den Entwicklungsprozess 
    beschleunigt.
    \item Eine automatische Versionierung veröffentlichter Änderungen, 
    die ein einfaches Zurücksetzen auf frühere Versionen ermöglicht.
    \item Eine kostenlose Hosting-Option mit einer Bandbreite von 
    bis zu 1 Gigabyte pro Monat, wodurch Kosten reduziert werden.
    \item Eine gute Integration mit externen Tools, wie beispielsweise 
    Voiceflow, was die Entwicklung zusätzlich unterstützt.
    \item Flexible Gestaltungsmöglichkeiten durch die Einbindung 
    eigener CSS-Styles.
\end{itemize}

Die Erfassung von Nutzerdaten stellt eine wichtige Grundlage 
für die Bewertung der entwickelten Funnel-Strategien dar. Ziel der 
Nutzerdatenerfassung ist es, das Verhalten der Nutzer:innen während der 
Interaktion mit den Websites zu analysieren und daraus 
Rückschlüsse auf die Benutzerfreundlichkeit sowie die Effektivität der 
jeweiligen Ansätze zu ziehen.

\setauthor{\firstauthor}
\subsection{Verwendete Tools}

Die gesammelten Daten stammen aus verschiedenen Quellen:
\begin{itemize}
    \item \textit{Framer Analytics}
    \item \textit{Microsoft Clarity}
    \item \textit{Google Analytics}
    \item \textit{Google Forms}
\end{itemize} 
Diese Tools ermöglichen eine umfassende Erfassung des Nutzerverhaltens.

Quantitative Daten werden hauptsächlich durch die folgenden Tools erfasst:
\begin{itemize}
    \item \textit{Framer Analytics}
    \item \textit{Microsoft Clarity}
    \item \textit{Google Analytics}
\end{itemize}

Qualitative Daten werden durch die folgenden Tools erfasst:
\begin{itemize}
    \item \textit{Google Forms}
    \item \textit{Microsoft Clarity}
\end{itemize}

Das in \textit{Framer} integrierte Analyse-Tool liefert grundlegende 
Kennzahlen wie Seitenaufrufe und Verweildauer und dient als erste 
Übersicht über das Nutzerverhalten direkt innerhalb der Entwicklungsumgebung.

Während \textit{Google Analytics} 
primär zur Erfassung von Kennzahlen wie Seitenaufrufen, Verweildauer auf
gewissen Unterseiten, Absprungraten und Conversion-Raten eingesetzt wird, 
bietet \textit{Microsoft Clarity} zusätzliche Einblicke in das konkrete 
Nutzerverhalten durch Heatmaps sowie visuelle Aufzeichnungen von 
Nutzerinteraktionen.

Ergänzend dazu wird \textit{Google Forms} verwendet, um 
subjektive Eindrücke der Testpersonen zu erfassen. Dadurch können 
qualitative Aspekte wie wahrgenommene Benutzerfreundlichkeit, Vertrauenswürdigkeit 
und Überzeugungskraft der beiden Funnel-Varianten erhoben werden. Die 
Kombination dieser Tools bildet die Grundlage für eine 
umfassende Analyse, da sowohl messbare Nutzungsdaten als auch 
persönliche Einschätzungen berücksichtigt werden.

%Nochmal drüberlesen und sicherstellen ob es stimmt
Die Erhebung der Daten erfolgt anonymisiert und ausschließlich zu 
Analysezwecken im Rahmen dieser Arbeit. Es werden keine personenbezogenen 
Daten gespeichert oder weiterverarbeitet. Dadurch wird sichergestellt, 
dass die Anforderungen des Datenschutzes eingehalten werden und die 
Privatsphäre der Nutzer:innen gewahrt bleibt.
Sämtliche Optionen in \textit{Google Analytics} und \textit{Microsoft Clarity} werden so
konfiguriert, dass keine personenbezogenen Daten erfasst oder gespeichert 
werden.

\subsection{Alternativen zu den verwendeten Tools}

Es gibt verschiedene Alternativen zu den in dieser Arbeit verwendeten 
Tools, die ebenfalls zur Nutzerdatenerfassung eingesetzt werden können. 
Dazu gehören:

\begin{itemize}
    \item \textit{Hotjar}
    \item \textit{Mixpanel}
    \item \textit{Matomo}

Diese Tools bieten ähnliche Funktionen wie die verwendeten Tools,

Absprungraten und Conversion-Raten. Zudem ermöglichen sie die Erstellung 
von Heatmaps und die Analyse von Nutzerinteraktionen.
Allerdings wurden die in dieser Arbeit verwendeten Tools aufgrund 
ihrer spezifischen Funktionen, Benutzerfreundlichkeit und der Möglichkeit, 
die erhobenen Daten effizient auszuwerten, ausgewählt. Insbesondere die 
Kombination von \textit{Google Analytics} und \textit{Microsoft Clarity} 
ermöglicht eine umfassende Analyse des Nutzerverhaltens, während 
\textit{Google Forms} Einblicke in die subjektiven Eindrücke 
der Nutzer:innen bietet.

Zudem sind die verwendeten Tools weit verbreitet und gut dokumentiert, was die
Integration und Nutzung erleichtert.

\setauthor{\firstauthor}
\subsection{Begründung der Toolauswahl}

Die Auswahl der verwendeten Tools erfolgte auf Basis funktionaler 
Anforderungen, Benutzerfreundlichkeit sowie der Möglichkeit, die 
erhobenen Daten effizient auszuwerten.

\textit{Google Analytics} wurde gewählt, da es eine etablierte und 
weit verbreitete Lösung zur Analyse von Webtraffic darstellt und 
umfassende quantitative Kennzahlen liefert. Dadurch eignet es sich 
besonders zur Messung von Conversion-Raten und zur Bewertung der 
Funnel-Performance.

\textit{Microsoft Clarity} ergänzt diese Daten durch detaillierte 
Einblicke in das tatsächliche Nutzerverhalten. Die Möglichkeit, 
Heatmaps sowie visuelle Aufzeichnungen von Nutzerinteraktionen zu analysieren, 
ermöglicht ein besseres Verständnis dafür, wie Nutzer:innen mit der 
Website interagieren und an welchen Stellen potenzielle Probleme auftreten.

Das integrierte \textit{Framer Analytics}-Tool wurde zusätzlich genutzt, 
da es eine einfache und direkte Analyse innerhalb der Entwicklungsumgebung 
ermöglicht. Dadurch können erste Erkenntnisse ohne zusätzliche Integration 
externer Tools gewonnen werden.

\textit{Google Forms} wurde eingesetzt, um qualitative Daten zu erheben, 
die durch Tracking-Tools allein nicht erfasst werden können. 
Insbesondere subjektive Eindrücke wie Vertrauen, Verständlichkeit und 
Überzeugungskraft lassen sich nur durch direkte Befragung der Nutzer:innen 
ermitteln.

Die Kombination der erhobenen Daten ermöglicht 
eine umfassende Betrachtung des Nutzerverhaltens, da sowohl 
quantitative als auch qualitative Daten in die Analyse einfließen. 
Dadurch kann eine objektive Bewertung und ein direkter Vergleich
der beiden Strategien vorgenommen werden.
